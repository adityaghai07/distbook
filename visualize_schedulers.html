Counting the bubble — 1F1B vs ZB1P vs DualPipeV

Four devices, six microbatches. Every device does the same 18 units of compute, so the idle bubble is just makespan − 18. Count the empty grid cells in any row to verify it.

units: forward F = 1 · input-grad Bi = 1 · weight-grad Bw = 1 · full backward B = Bi+Bw = 2
model: instant communication · greedy list schedule (each device runs its earliest-ready op) · p = 4, n = 6, W = 1
forward (F) backward — full B, or input-grad Bi weight-grad (Bw) · filler dashed = counter-stream (mb 4–6) empty cell = idle bubble
The exposed bubble on the critical device is the backward return trip: (p−1) hops, each costing the per-hop return latency. 1F1B returns a full backward (2) per hop → (p−1)·(F+B) = 9. ZB1P returns only the input-grad (1) and defers weight-grad to fillers → (p−1)·(F+B−2W) = 3. DualPipeV runs a second stream from the opposite end whose forwards drop straight into the return-path gap → ≈ 1. Click any cell to trace a microbatch across devices.
